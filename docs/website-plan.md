# SaveWatt Website Plan

> **Builder entry point:** Start with `docs/BUILD-HANDOFF.md`. This file remains
> the detailed sitemap and page-role source.

## Scope

Phase 1 is a planning round.
This document defines the complete website structure that the next swarm will build from.

## Site Goal

Build a compact but scalable French website centered on one high-converting landing page, supported by trust, sector, service, and legal pages.

## Landing Page Role

The home page is the main commercial page.
It should also behave like the core landing page for paid, outbound, and organic traffic.

## Landing Page Section Order

1. Hero
2. Reassurance strip
3. Problem / `Le constat`
4. Solution / `Notre solution`
5. Process / `Comment ca marche`
6. Audience / `A qui s'adresse SaveWatt`
7. Why SaveWatt
8. Results / proof
9. FAQ / objections
10. Final CTA

## Page Role Summary

### `/`

- Full conversion page
- Best first-touch page for most traffic
- Carries the strongest copy, proof, and visual system

### `/solution`

- Cleaner narrative version of the offer
- Better for detail-seeking visitors and outbound links

### `/methodologie`

- Process, what documents are needed, what happens next
- Reduces friction and uncertainty

### `/services`

- Service overview hub
- Future expansion point for deeper service detail pages

### `/secteurs`

- Sector hub to qualify demand
- Links to high-priority target segments

### `/resultats`

- Proof and ROI framing
- Example savings tables, disclaimers, methodology notes

### `/pourquoi-savewatt`

- Independence, success-fee model, reporting, confidentiality, regulatory expertise

### `/faq`

- Pre-conversion objection handling

### `/contact`

- Audit request flow and contact reassurance

## Policy And Support Pages

The website should launch with:

- `mentions-legales`
- `politique-de-confidentialite`
- `cookies`
- a clear contact page
- a simple FAQ

These are not optional.
They are part of the trust layer for a business handling invoices, contracts, and organization data.
