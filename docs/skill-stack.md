# SaveWatt Skill Stack

## Purpose

This document assembles the skill stack for the SaveWatt project with one primary objective:

Build a distinctive, conversion-focused French landing page and the planning documents that support it:

- `docs/vision.md`
- `docs/branding.md`
- `docs/landingpageplan.md`
- `docs/competitor-analysis.md`
- `docs/seo-plan.md`
- `docs/visual-dna.md`
- `docs/copy-platform.md`

## Current Context

- Workspace currently contains the SaveWatt PDF source material and no existing product architecture files.
- `ARCHITECTURE.md` is not present yet, so this plan treats the project as a greenfield marketing-site foundation.
- Source inputs available now:
  - SaveWatt brochure copy
  - SaveWatt visual direction from the provided screenshot
  - Competitor target URL: `https://www.energie2france.fr/`

## Core Skill Stack

These are the skills we should actively use for this project.

### 1. Orchestration And Planning

- `find-skills`
  - Use first whenever we need to expand the skill set or verify if a stronger specialist skill exists.
- `design-brief`
  - Use to frame the project problem, audience, constraints, and success metrics.
- `design-principles`
  - Use to define the non-negotiable rules for SaveWatt’s landing page quality and brand behavior.
- `paseo-orchestrate`
  - Use to coordinate multi-step execution across docs, design, copy, SEO, and implementation.

### 2. Brand Strategy And Brand Guidelines

- `branding`
  - Owns positioning, values, narrative, voice, tone, differentiation, and messaging hierarchy.
- `brand-visual-generator`
  - Owns visual identity, typography, color system, spacing system, and design tokens.
- `design-dna`
  - Use to extract and formalize SaveWatt’s design language from the screenshot, brochure, and future references into a repeatable system.

### 3. Taste And Premium Frontend Design

- `design-taste-frontend`
  - Primary taste skill for anti-generic, high-agency interface design.
- `high-end-visual-design`
  - Primary “impeccable Awwwards-tier” design skill for premium layout, typography, and visual rhythm.
- `gpt-taste`
  - Use when we want stronger layout variance, richer section choreography, and premium editorial pacing.
- `stitch-design-taste`
  - Use if we want to generate a reusable `DESIGN.md` design system artifact for AI-assisted screen generation.
- `interaction-design`
  - Use for state design, UX feedback, and intentional interface behavior.
- `design-critique`
  - Use to review the page direction before implementation hardens.
- `design-qa-checklist`
  - Use during final implementation review to ensure the build matches the intended design system.

### 4. Motion And Awwwards-Level Execution

- `awwwards-animations`
  - Primary motion skill for premium scroll choreography, reveal systems, and hero interactions.
- `animation-principles`
  - Use to keep motion purposeful, performant, and not decorative noise.
- `lottie-animations`
  - Use only if we decide specific vector loops or explainer motion are better than code-driven animation.

### 5. Landing Page Strategy And Conversion

- `landing-page`
  - Owns the single-offer page structure, section hierarchy, and conversion path.
- `landing-page-optimization`
  - Owns offer clarity, friction reduction, CTA logic, proof placement, and conversion improvement.
- `hero-generator`
  - Owns the above-the-fold concept, messaging compression, hero composition, and CTA framing.
- `analytics-tracking`
  - Owns measurement of CTA clicks, form submissions, scroll depth, and conversion attribution.

### 6. Copywriting And Marketing Psychology

- `copywriting`
  - Use for headlines, CTA lines, section copy, and concise landing page messaging.
- `copywriting-core`
  - Use for deeper messaging passes, editorial tightening, and higher-discipline conversion writing.
- `marketing-psychology`
  - Use to structure persuasion ethically: risk reversal, trust, proof, framing, urgency, and clarity.
- `content-strategy`
  - Use to align landing page messaging with the future SEO and content ecosystem.
- `copy-editing`
  - Use for final polish once the core messaging is approved.

### 7. Competitor Research And Positioning

- `competitor-analysis`
  - Primary strategic competitor skill for positioning, SWOT, comparison grids, and market framing.
- `competitor-profiling`
  - Use for deep URL-based competitor dossiers, especially for `energie2france.fr`.
- `competitor-research`
  - Use for keyword, backlink, content-gap, and SEO-level competitor research.
- `seo-competitor-pages`
  - Use if we later create SEO comparison pages or “alternative to” content.

### 8. SEO, AEO, And Organic Visibility

- `seo-strategy`
  - Owns prioritization and roadmap order for organic growth.
- `seo-and-aeo-strategy`
  - Owns classic SEO plus AI visibility, answer engine readiness, and citation-friendly structure.
- `seo`
  - Use for broader audits once a live site exists.
- `seo-technical`
  - Use for technical health, Core Web Vitals, crawlability, and rendering checks.
- `seo-schema`
  - Use for structured data and rich-result readiness.
- `seo-content`
  - Use for E-E-A-T, clarity, depth, and citation-worthiness.
- `local-seo`
  - Use because SaveWatt is a Paris-based service and local trust signals matter.
- `entity-seo`
  - Use to strengthen brand/entity clarity around SaveWatt, energy optimization, and geographic trust.

### 9. Visual Assets, Image Direction, And Prompting

- `image-taste-frontend`
  - Use to define image style that matches the page instead of fighting it.
- `visual-dna-assets`
  - Primary skill for building the visual DNA, asset register, paste-ready prompts, and localization-aware image production pipeline.
- `brand-visual-generator`
  - Use again here for prompt constraints tied to brand identity.
- `visual-content`
  - Use for conceptualizing supporting visuals, diagrams, explainer imagery, and social derivatives.
- `image-optimization`
  - Use during implementation to preserve quality without hurting performance.
- `seo-images`
  - Use once assets exist to make images search-friendly and semantically useful.

### 10. Quality, Accessibility, And Validation

- `accessibility-audit`
  - Required before final launch review.
- `accessibility-test-plan`
  - Use to define the manual and automated checks for the page.
- `design-qa-checklist`
  - Required for design fidelity validation after implementation.
